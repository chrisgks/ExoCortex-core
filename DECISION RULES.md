# Root Decision Rules

Standing rules and routing guardrails for this instance. This starts
minimal on purpose; add rules as patterns prove out. Durable changes go
through the review loop, never silently.

## Routing

- Match the task to the nearest scope's `AGENT.md` stance, then to a skill
  by its "use this when" description. You can always name a skill outright
  to skip the matching.

## Promotion

- Keep new signal as a candidate until reviewed. Promote to memory, a rule,
  a workflow, or a skill only after evidence accumulates.
