Compounding Loop

Work compounds because state lands in files.

The system is not the chat window. The system is the repeatable loop that starts from context, uses stable roles and tools, and writes reusable artifacts back into the repo.
Founding Constraint

Future work should start from something better than zero, without hiding the operating state inside opaque memory.

Step 1

Start in a real folder

The path is the first routing decision. It establishes the boundary and determines which truth surface should dominate.

Step 2

Load contracts and defaults

The wrapper assembles local and ancestor context, then resolves agent and mode rather than improvising each time.

Step 3

Apply role, rules, skills

Specialization comes from composition: stable agent + local rules + reusable methods + tools.

Step 4

Do ordinary work

Use shell commands, scripts, wrappers, review loops, Mission Control, or external CLIs without making them the source of truth.

Step 5

Write durable artifacts

Sessions generate inspectable summaries, candidates, journal entries, wiki synthesis, and promotable signal.

Step 6

Reuse later

The next session sees a better starting state because context, workflows, and decisions now exist as files.

Why This Matters
Chat does not become the system.

You can switch harnesses or providers because the operating surface is local and file-based.

Sessions stop disappearing.

Useful patterns can survive the run instead of dying with the transcript scrollback.

Improvement becomes inspectable.

The repo can evolve through diffs, review, and promotion rather than vague claims about “memory.”
