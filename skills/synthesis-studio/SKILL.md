# Synthesis Studio

Pull scattered work — across projects, repos, and notes — into one place and turn
it into proposals someone can act on. The process below is a sensible default;
scale the depth to the size of the job.

## When to use / not to use

- Use it when real work is spread across several places and the goal is a consolidated plan, not a one-off summary.
- Don't use it for a single-source question (answer inline) or when the work already lives in one place.

## The process

### 1. Map the sources
Ask where the relevant work actually lives — get specific paths, not guesses.
List each source: what it is, where it is, and whether it's still active.

### 2. Capture each source
Read each source properly before summarizing it. Write a short note per source:
what it is in one line, the substance worth keeping, and what it adds to the
whole. Number the notes so the read order is obvious later.

### 3. Cluster by theme
Once the sources are captured, group what recurs across them by theme — not by
which source it came from. The clusters are the real structure; the source
boundaries usually aren't.

### 4. Find the gaps
Look at the seams between clusters — where the work points somewhere but nobody
has connected it yet. The strongest proposals usually live in those gaps, not
inside any single cluster.

### 5. Pick a few, sharply
Choose a small number of concrete options from the gaps — three or so, not
fifteen. For each: what it is in a paragraph, why it's worth doing, the cheapest
way to test whether it works, and what it unlocks if it does.

### 6. Write the proposals
Turn each option into a short plan tuned to its audience — what it is, what it
costs, what decides it, and the next step. One document per audience, or one
document with per-audience summaries.

### 7. Record what's durable
Promote findings that will outlast this job into your knowledge base, one per
idea, with a pointer to where it came from. That makes the next consolidation
cheaper.

## Guardrails

- Cite where each finding came from.
- Human-in-loop: propose the plan, don't decide it.
