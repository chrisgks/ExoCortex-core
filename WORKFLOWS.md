# Root Workflows

## Best Next Action

1. Read root `STATE.md`, `system/PRIORITIES.md`, and `system/OPEN LOOPS.md`.
2. Read the relevant domain or project `STATE.md`.
3. Identify the highest-leverage unresolved action.
4. Expand the decision space briefly when doing so improves the decision.
5. Choose one next action under current constraints.
6. Produce one concrete next step with a done condition.
7. Route execution to the right agent if needed.

## Purpose-Aligned Capture

1. Treat explicit preferences, role/context updates, recurring decision patterns, and operating expectations as high-signal.
2. Decide the correct destination: `MEMORY.md`, `system/SELF MODEL.md`, `system/DECISION RULES.md`, a `skill`, or the relevant `wiki/` page.
3. Capture immediately only when the user states a durable correction or explicit operating instruction.
4. Queue rather than promote when the signal is inferred, emotionally state-dependent, or contradicted by existing context.
5. Preserve concise rationale when updating durable state.

## Project Intake

1. Identify domain and project from the path.
2. Read the nearest `AGENT.md`, `README.md`, and `STATE.md`.
3. State current scope, likely agent, and expected output.
4. If no project exists, scaffold one before storing project-specific memory.

## Review

1. Treat review as bug/risk discovery first.
2. Lead with findings, ordered by severity.
3. Include file and line references.
4. Keep summaries secondary.
5. Capture remaining uncertainty as open questions or coverage gaps.
